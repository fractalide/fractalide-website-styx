Hyperflow

Hyperflow is a browser for smart contracts apps and cryptocurrencies

It draws much inspiration from HyperCard, by allowing non-technical people to easily build and run problem solving apps. Though we will differ from HyperCard on these points.

Hypercard concepts to keep

The intuitive, and easy to learn approach HyperCard is famous for.

It’s very easy to try out your newly built app, by simply switching between run mode and design mode.

Keep the concept of cards and stacks, an analogy would be; a card is a webpage and a stack is a website. Each card contains GUI components and logic that links other cards together to form a stack of cards.

Cards, stacks and low level components are shareable via Fractalmarket.

Hypercard concepts to remove

Remove the HyperTalk programming language and replace it with Flowscript, a Flow-based Programming language that promotes reusability.

HyperCard didn't have a network layer, it should go without saying, but this will be corrected.

HyperCard is old and outdated. Hyperflow will be a fresh new take on HyperCard, being completely rewritten from scratch

Node mode

Node mode of development is primarily for programmers, allowing them to build reusable, reproducible and composable software agents or “black box” processes.

Racket, the programming language used, gives programmers a wealth of programming language concepts to use at their disposal.

Graph mode

Graph mode allows domain experts, such as engineers, biologists and mathematicians etc, to compose agents created by programmers into networks of nodes. These networks of “black box” processes exchange data across predefined connections by message passing, where the connections are specified externally to the processes. These “black box” processes can be reconnected endlessly to form different applications without having to be changed internally.

Run mode

Run mode allows all users to execute applications created by programmers and domain experts. This is the equivalent mode to a traditional HTTP browser.
